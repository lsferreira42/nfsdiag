NFSDIAG(1) General Commands Manual NFSDIAG(1)

NAME

nfsdiag — diagnose NFS servers from the client side.

SYNOPSIS

nfsdiag [OPTIONS] <server-ip-or-hostname>

DESCRIPTION

Walks the NFS stack from the wire up: TCP reachability, rpcbind, NFS v2/v3/v4/v4.1/v4.2 probes, mountd, exports, mount cascade, permissions, root squash, ACLs, locking, stale handles, performance.

One IP, one command. Compact text by default. JSON and standalone HTML when you need it. Safe by default — validated inputs, trusted helper execution, automatic mount namespace, O_NOFOLLOW reports, and opt-in dangerous probes.

Written in C99 against libtirpc. MIT. Current version: 0.10.0.

Created by Leandro Ferreira — see more on the author page.

# what it checks

network rpcbind tcp/111 · NFS tcp/2049 · IPv4/IPv6 with fallback · optional UDP probe
rpc service map from rpcbind · mountd v1/v2/v3 · NFS v2/v3/v4/v4.1/v4.2 NULLPROC probes · lockd/NLM · statd/NSM
mount v4.2 → v4.1 → v4 → v3 cascade · effective options parsed from /proc/self/mountinfo
fs close-to-open consistency · read · write · fsync · directory listing · copy_file_range · fallocate · O_DIRECT
acl POSIX ACLs · NFSv4 ACLs · generic xattrs · SELinux contexts
perm UID/GID/supplemental groups simulation · practical root_squash detection
lock fcntl advisory locks · NLM · NSM
perf throughput smoke test · metadata latency · external fio backend · deep metrics from /proc/self/mountstats
stale ESTALE loop · pNFS layouts · NFSoRDMA hints
filenames 255-byte names · spaces · colons · UTF-8 multibyte · per-rejection errno
delegation NFSv4 delegation activity via /proc/self/mountstats (DELEGRETURN operations)
report tagged text · UTF-8 table · NDJSON · Prometheus · JSON · HTML · evidence bundle · stable check IDs · remediation text

# usage

full diagnostic — needs root for mount

sudo nfsdiag 192.168.1.10

network and RPC only — no mount, no root

nfsdiag --no-mount 192.168.1.10

simulate a specific identity (UID/GID + groups)

sudo nfsdiag --uid 1000 --gid 1000 --groups 10,20 192.168.1.10

machine-readable report for CI / scripts

nfsdiag --json=report.json 192.168.1.10

production audit — read-only, dry run, rate-limited

nfsdiag --dry-run --read-only --delay-ms 500 prod-nfs.example.com

shareable HTML for a support ticket

sudo nfsdiag --html=report.html 192.168.1.10

continuous monitoring — re-run every 60 s

sudo nfsdiag --watch 60 192.168.1.10

stream failures with jq

sudo nfsdiag --output-format=ndjson \
  192.168.1.10 | jq 'select(.level=="fail")'

safe preset — read-only diagnostics

sudo nfsdiag --profile safe 192.168.1.10

evidence bundle for tickets

sudo nfsdiag --output-dir ./nfsdiag-report 192.168.1.10

compare before / after reports

nfsdiag diff before.json after.json

audit a fleet from a hosts file

sudo nfsdiag \
  --hosts-file /etc/nfs-servers.txt \
  --json=audit.json
